What are the benefits of the IPBee Certificate of Evidence?

The client receives evidence that is unique in its form and cannot be manipulated according to the current state of the art. It can be used and verified worldwide. This innovative solution is admitted as evidence in court in Asia (netcourt).

The proof of IPBee creates trust and reduces discussion and/or eliminates disputes.

What does IPBee confirm with this service?

IPBee confirms the existence of a digital document and its content at the time stated in the certificate. This can be used as evidence for the authenticity of copyrights, trade secrets, certificates, invoices, documentation of prior use, R&D status, business correspondence, etc.

What data will be transferred to IPBee using the HASH generator?

Only cryptographic code (HASH) and file name are transmitted to IPBee.

Is the service via HASH generator GDPR compliant?

Yes. We only receive cryptographic codes of the original file. A recovery of the file is not possible (it is virtually a one-way street).

Therefore, we can guarantee absolute GDPR compliance, which means that noconclusion of a GDPR agreement is required.

Is the HASH generator suitable for proof of trade secrets?

Yes, the files and their contents are not passed on to us via the HASH generator but remain exclusively with you. The necessary HASH is calculated directly in your browser on your PC. We only receive the HASH and the file name. You can use our generated proof with the included certificate anytime to prove the existence of the file and its content at the stored and indicated time.

What does HASH mean, what is its function and use?

A HASH is usually a cryptographic procedure which is used to uniquely describe the contents of a file. Any change in the content of the original file automatically results in a complete change of the HASH code. It is virtually impossible to find two different files with the same exact HASH code.

What HASH methods does IPBee use?

IPBee uses HASH SHA 256 for all files.

Various sources, including Wikipedia, provide information about this.

For graphic files (copyrights), we use another method for our clients, the 4D Security HASH. This encodes metadata, content, raw data similarity and raw data integrity.

How long does it take for my files to be protected?

After starting the process at IPBee, the first protection currently takes about 3-5 minutes. Full protection in currently all three blockchains can take up to 24 hours.

What do I get from IPBee?

Upon completion of the IPBee process, you will receive a certificate that contains a description, all essential information about the original file (if available), the transactions and storage in the different blockchains as well as a short verification guide.

What is the easiest way to perform the authentication check?

Simply upload the original file to the homepage https://blockchain.ip-bee.de/validierung/ and compare the data on the screen with those on the certificate.

Alternatively click on the link of the HASH(es) in the certificate to compare the values.

What to do if the check results differ or do not return any results at all?

This can have various reasons.

Do you have the certificate matching the original file?

If so, the following could apply:

a) HASH value original application file in the certificate does not match the currently calculated HASH value.

Did you select the correct calculation method in HASH-maker?

Have you checked it on our homepage by uploading the file?

If values do not match, you do NOT have the original document.

b) The digital signature of the IPBee certificate is "compromised”

The signature display in the pdf-reader shows a change in the file after we have signed it (qualified digital signature).

This is probably an attempt to manipulate our proof.

Please upload the original file in the validation area. We will tell you whether we know the file (HASH) and whether we have performed a successful blockchain protection for it and for whom we have performed it.

c) Transaction IDs show no (successful) transactions in the blockchain

If the servers are reachable but no suitable transactions can be found (possibly manually and not in our validation area), then a final check of the stored PRIVATEKEYS or HASHES in the blockchains cannot take place. The proof of authenticity cannot be provided.

In case of doubt, please contact us at info@ip-bee.de.

Not all blockchain transaction codes are visible on the certificate. What does this mean?

For various reasons, it can happen that not all transactions are executed (on time). For this reason, we use several blockchains to ensure that our clients always have at least one protection.
